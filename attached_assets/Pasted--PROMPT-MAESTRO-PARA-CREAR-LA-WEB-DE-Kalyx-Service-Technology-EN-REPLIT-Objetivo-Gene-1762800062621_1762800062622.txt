# PROMPT MAESTRO PARA CREAR LA WEB DE **Kalyx Service Technology** EN REPLIT

> **Objetivo**: Generar una web profesional, moderna y de alto rendimiento para la empresa *Kalyx Service Technology* (desarrollo de software y consultoría), lista para publicar en Replit. Debe incluir diseño, copy en español (con opción de inglés), navegación, SEO técnico, formularios funcionales, blog, CMS simple basado en archivos, y componentes reutilizables. Entrega en un solo repositorio con instrucciones de despliegue.

---

## Rol / Modo de trabajo del generador

Actúa como **Product Designer Senior + Tech Lead Full‑Stack**. Toma decisiones de diseño coherentes, documenta tus elecciones y entrega una solución **pixel-perfect** y **production‑ready**.

---

## Stack y requisitos técnicos

* **Framework**: React + Vite (o Next.js si lo consideras superior). Si usas Next.js, habilita SSG/ISR.
* **UI**: Tailwind CSS + shadcn/ui + lucide-react.
* **Accesibilidad**: WCAG 2.2 AA (focus states visibles, semántica correcta, contrastes ≥ 4.5:1, skip links).
* **Rendimiento**: Lighthouse ≥ 95 (Performance/SEO/Best Practices/Accessibility). Imágenes optimizadas (next/image o vite-imagetools), carga diferida, prefetch.
* **i18n**: ES y EN (estructura de traducciones por JSON).
* **Contenido**: Markdown para blog y páginas largas (MDX opcional). CMS ligero por archivos (`/content`).
* **Formularios**: Envío vía API interna (Node/Express) o servicios tipo Formspree; incluye validación Zod + react-hook-form, honeypot y rate limit básico.
* **Analytics**: Consent mode + integración (p.ej. Plausible o Google Analytics) activable por env var.
* **Seguridad**: Headers estrictos (CSP no-blocking para fuentes/analytics, HSTS, no sniff, XSS protection), sanitizado de entradas, CSRF en endpoints.
* **DevOps**: Scripts npm (`dev`, `build`, `preview`, `start`). Archivo README con pasos de despliegue en Replit.
* **Testing**: Playwright o Vitest + Testing Library para componentes críticos (Navbar, CTA, Form contacto).

1. **Home**

2. **Servicios** (landing general + subpáginas):

   * **Descubrimiento & Product Strategy**: discovery sprints, mapa de stakeholders, roadmap, Product/Market Fit.
   * **Diseño UX/UI & Design Systems**: research, wireframes, prototipos, DS con tokens y librería de componentes.
   * **Desarrollo Web & APIs**: front (React/Next), back (Node/Nest), APIs REST/GraphQL, microservicios, WebSockets.

   * **Mantenimiento & Soporte**: SLAs, monitoreo, rotación on‑call.

4. **Casos de Estudio**: 3–5 casos con contexto, hipótesis, solución, arquitectura, métricas, lecciones.

5. **Precios**: Paquetes (Discovery Sprint, MVP Sprint, Escalamiento, Soporte) + precios “desde”, comparativa y preguntas frecuentes. Nota legal.

6. **Stack & Prácticas**: toolchain, estándares de código, seguridad, performance, accesibilidad.

7. **Blog/Insights**: artículos MDX (guías técnicas, estrategia producto, estudios de performance). Buscador y categorías.

8. **Nosotros**: misión, visión, valores, equipo (cards con social), cultura de trabajo.

9. **Carreras**: roles, proceso de selección, formulario de aplicación.

10. **Contacto**: formulario (nombre, email, empresa, tamaño, presupuesto, mensaje), link a Calendly opcional, datos legales.

11. **Legal**: Privacidad, Términos, Cookies (banner con preferencias), DPA template.
